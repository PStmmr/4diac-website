Developing Function Blocks

This section moves from the System perspective to the Type Management perspective . Here the interface and the internals of Function Block types can be seen and new FBs can be created and exported to the FORTE. The new FBs are then available in the FB palette for use in the Systems' Applications.

This section covers how to create your own function block types. New function blocks need to be created when the needed functionality doesn't exist or multiple FBs should be combined into just one FB. First you have to define the interface with its input and output events and data. And then define the FB internals, which are defined by the kind of FB. We implement the Flip-Flop functionality inside all three different FB types to introduce their creation process.

The initial creation of all FB types is the same. Within the Type Management Perspective create a new folder within your System. Then create a New Type. Within the Wizard select the desired System and the folder you created, add your FB's Type Name like FlipFlop, choose the desired Type and press Finish.

Then create your desired interface and follow the instructions of your chosen FB type.

  • Flip-Flop as Basic FB (BFB)
  • Flip-Flop as Composite FB (CFB)
  • Flip-Flop as Service Interface FB (SIFB)

After you have finished the design of your Function Block you have to export it for FORTE that it can be used within your Application. Once you have successfuly compiled your Function Block, you are able to test it with 4DIAC's Function Block tester.
