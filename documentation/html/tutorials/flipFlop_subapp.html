Create a Subapplication

A Subapplication is used to group a part of an Application that belongs together. It appears like 1 FB where inputs and outputs have the real FB instance name as prefix. Inputs and outputs with no direct connection to outside can be hidden. Subapplications can be mapped to the same hardware like a FB but its internally FBs can also be mapped to different hardware (Devices or Resources). Subapplications can contain other Subapplications.

Create a Subapplication

Select the blocks to go into the desired Subapplication. If mapped to any hardware, unmap now. Right click and chose New Subapplication. The FBs will now appear as if they are within one FB. Give the SubApp a better instance name

Subapplication Network

Double click on the SubApp to open a window with the Subapplication contents of its function block network.

Subapplication Interface

In the Application window select the SubApp block. In Properties window at the bottom click on Interface. Inputs/Outputs with no interface outside the SubApp are just checked whereas an interface outside the SubApp are filled-in. Un-check all inputs/outputs just checked, so we can see the interface we use for the SubApp.

Now the SubApp interface we use is shown and can be mapped to the desired hardware.
